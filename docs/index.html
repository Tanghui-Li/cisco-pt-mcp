cisco-pt-mcp.

Let AI clients control a running Cisco Packet Tracer instance.
Powered by MCP, Socket.IO, and a lightweight PT extension.

Topology Builder

Build complete networks with a prompt.

Add routers, switches, and PCs to the canvas, connect them with the right cable type, and configure IPs — all from a single AI conversation.

  • 15+ device types — routers, switches, PCs, servers
  • Automatic or explicit cable selection
  • Coordinate-based placement on the canvas
  • Interface module installation at runtime
AI > addDevice("R1", "Router-PT", 100, 200)
{ "success": true }
AI > addDevice("SW1", "2960-24TT", 300, 200)
{ "success": true }
AI > addLink("R1", "Gig0/0", "SW1", "Fa0/1", "auto")
{ "success": true, "message": "Link created" }
IOS Configuration

Configure any IOS device.

Run multi-line IOS CLI commands on any router or switch — routing protocols, VLANs, ACLs, NAT — and have the config written to memory automatically.

  • Full IOS command support, any mode
  • Multi-line config blocks in one call
  • write memory runs automatically after config
  • PC IP, DHCP, gateway, and DNS via a single tool
AI > configureIosDevice("R1",
"interface Gig0/0\n ip address 192.168.1.1 255.255.255.0\n no shutdown\nip route 0.0.0.0 0.0.0.0 10.0.0.1")
{
"success": true,
"message": "4 commands applied"
}
Simulation

Simulate and verify, hands-free.

Switch to simulation mode, send PDUs between any two devices, step through frames, and inspect results — without touching the mouse.

  • ICMP, TCP, UDP, HTTP, HTTPS, DNS PDU types
  • Step forward, backward, or reset the timeline
  • Inspect accepted, dropped, and in-transit frames
  • Full command log retrieval per device
AI > sendPdu("PC0", "Server1")
{ "success": true }
AI > stepSimulation("forward", 5)
{ "frameCount": 12 }
AI > getPduResults(["ICMP"])
{ "frames": [{ "status": "accepted" }] }

Install.

Two components. Five minutes. Works with Claude, Cursor, VS Code, and more.

1 — MCP Server

First install uv if you don't have it, then register with your client:

Terminal
claude mcp add cisco-pt-mcp --scope user -- uvx cisco-pt-mcp
%APPDATA%\Claude\claude_desktop_config.json
{
  "mcpServers": {
    "cisco-pt-mcp": { "command": "uvx", "args": ["cisco-pt-mcp"] }
  }
}
%USERPROFILE%\.cursor\mcp.json
{
  "mcpServers": {
    "cisco-pt-mcp": { "command": "uvx", "args": ["cisco-pt-mcp"] }
  }
}
.vscode/mcp.json
{
  "servers": {
    "cisco-pt-mcp": { "command": "uvx", "args": ["cisco-pt-mcp"] }
  }
}
%USERPROFILE%\.codex\config.toml
[mcp_servers.cisco-pt-mcp]
command = "uvx"
args = ["cisco-pt-mcp"]
Terminal
backboard config add-mcp "{\"name\": \"cisco-pt-mcp\", \"command\": \"uvx\", \"args\": [\"cisco-pt-mcp\"]}"

2 — Packet Tracer Extension

Download cisco-pt-mcp.pts and install via Packet Tracer's Extension Manager:

  1. Open Packet Tracer.
  2. Go to Extensions → Extension Manager → Install Extension…
  3. Select the downloaded cisco-pt-mcp.pts file.
  4. Restart Packet Tracer when prompted.
  5. Click Extensions → Packet Tracer MCP to open the bridge window.

3 — Run

Open the bridge window in Packet Tracer, then start your MCP client. The status pill flips to connected within a second. Start prompting.
