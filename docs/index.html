9 tools exposed

Everything the model needs to build and inspect a full Packet Tracer lab.

addDevice
Drop a router, switch, or PC onto the canvas at a given position.
addModule
Install an interface module into a device slot (e.g. add a WIC card to a router).
addLink
Connect two devices with a cable — copper straight-through, crossover, serial, fiber, and more.
removeDevice
Delete one or more devices from the topology by name.
removeLink
Delete one or more cables identified by device and port name.
configurePcIp
Set static IP, subnet mask, default gateway, and DNS — or enable DHCP — on a PC.
configureIosDevice
Run any IOS CLI commands on a router or switch. Multi-line configs supported.
getNetwork
Snapshot of all devices, their interfaces, and all cable connections in the current workspace.
getDeviceInfo
Detailed view of a single device — model, interfaces, and connected neighbours.

Install

1 — MCP Server

First, install uv if you don't have it (one command on any platform). Then register the server with your MCP client:

Terminal
claude mcp add cisco-pt-mcp --scope user -- uvx cisco-pt-mcp
%APPDATA%\Claude\claude_desktop_config.json
{
  "mcpServers": {
    "cisco-pt-mcp": { "command": "uvx", "args": ["cisco-pt-mcp"] }
  }
}
%USERPROFILE%\.cursor\mcp.json
{
  "mcpServers": {
    "cisco-pt-mcp": { "command": "uvx", "args": ["cisco-pt-mcp"] }
  }
}
.vscode/mcp.json
{
  "servers": {
    "cisco-pt-mcp": { "command": "uvx", "args": ["cisco-pt-mcp"] }
  }
}
%USERPROFILE%\.codex\config.toml
[mcp_servers.cisco-pt-mcp]
command = "uvx"
args = ["cisco-pt-mcp"]
Terminal
backboard config add-mcp "{\"name\": \"cisco-pt-mcp\", \"command\": \"uvx\", \"args\": [\"cisco-pt-mcp\"]}"

2 — Packet Tracer Extension

Download cisco-pt-mcp.pts then install it through Packet Tracer's Extension Manager:

  1. Open Packet Tracer.
  2. Go to Extensions → Extension Manager → Install Extension…
  3. Select the downloaded cisco-pt-mcp.pts file.
  4. Restart Packet Tracer when prompted.
  5. Click Extensions → Packet Tracer MCP to open the bridge window.

3 — Run

Launch Packet Tracer and open the bridge window. Then start your MCP client — the bridge status pill flips to connected within a second. Start prompting.
